巡检系统
项目简介：智能巡检管理系统
一、 项目概述
本项目旨在构建一套智能化、可视化的巡检管理系统，通过整合移动端（App）与Web端（浏览器），实现对巡检人员、巡检路线及安全隐患的全流程闭环管理。系统以“风险”为核心数据对象，利用实时地图技术与数据库记录机制，打通一线巡检员、后台管理员及访客（监管层）之间的信息壁垒，确保巡检工作高效、透明、可追溯。
二、 核心功能模块
实时地图监控（可视化模块）
巡检员动态追踪：在Web端地图仪表盘上实时显示巡检员的当前位置及移动轨迹。
关键点管理：地图上清晰标注巡检路线中的必经检查点，支持巡检员在检查点进行“打卡”操作，确保巡检路径无遗漏。
风险事件闭环管理（数据库模块）
风险上报（巡检员端）：巡检员在巡检过程中发现安全隐患（风险），通过手机App进行拍照上传，并关联风险等级（低危/中危/高危）、地理位置及文字描述，数据自动存入数据库。
处置与归档（管理员端）：管理员通过浏览器后台查看风险数据库，对上报的风险进行核实、指派处理，并在问题解决后标记为“已解决”进行归档，形成“上报-处置-归档”的闭环。
多角色协同工作流
巡检员（移动端）：登录账号后按预定路线执行巡检任务，通过“打卡”确认点位巡查，遇风险即拍即传。
管理员（Web端）：登录后台管理系统，负责巡检计划的制定、风险数据库的审核与状态更新，以及人员管理。
访客（Web端）：无需登录或通过特定权限访问地图仪表盘，可直观查看区域地图、当前巡检路线、在岗巡检人员位置及历史风险点分布情况，实现监管与信息透明。
三、 项目价值
该系统通过数字化手段规范了巡检作业标准，解决了传统巡检中“漏检、错检、信息传递滞后”的痛点。实时地图功能提升了应急调度能力，而风险数据库的建立则为安全管理提供了数据决策支持，有效提升了整体区域的安全防控水平。
术语：
风险：存在安全隐患的地点等，一般由一个地址，示意图，描述，风险等级（低危，中危，高危），解决状态（未解决，已解决）组成
巡检：在规定路线进行巡逻和记录风险的过程
功能部件：1.实时地图（巡检员实时位置，巡检员检查点）2.事件记录/解决数据库（巡检员记录安全风险，管理层标记解决）
运行流程：
巡检员：使用手机app，登录巡检员账号，在检查点打卡，按照预定路线巡检（动态在网络实时地图显示位置），遇到风险拍照并进行风险归类后上传数据库
管理员：使用浏览器，登录管理员账号，检查风险数据库，进行解决后归档
访客：通过浏览器访问网站地图仪表盘（有以下信息：区域地图，巡检路线，巡检人员，风险点）

地理数据库：PostgreSQL + PostGIS
通信：Socket.IO（实时） REST api（提交）

四、API文档（第一阶段）
基础信息
- Base URL：`http://localhost:8081`
- 数据格式：`application/json`
- 认证方式：基于 Session（`your_app_session` Cookie）

1. 登录
- 方法：`POST`
- 路径：`/login`
- 请求体：
```json
{
	"username": "admin",
	"password": "admin123"
}
```
- 成功响应（200）：
```json
{
	"success": true,
	"user": {
		"id": 1,
		"username": "admin",
		"roles": "admin",
		"EmployeeID": "1"
	}
}
```
- 失败响应（401）：
```json
{
	"error": "用户名或密码错误"
}
```

2. 退出登录
- 方法：`POST`
- 路径：`/logout`
- 说明：需已登录
- 成功响应（200）：
```json
{
	"success": true
}
```

3. 获取当前登录用户
- 方法：`GET`
- 路径：`/me`
- 说明：需已登录
- 成功响应（200）：
```json
{
	"user": {
		"id": 1,
		"username": "admin",
		"roles": "admin",
		"EmployeeID": "1"
	}
}
```

4. 新增用户（仅管理员）
- 方法：`POST`
- 路径：`/users`
- 权限：`admin`
- 请求体：
```json
{
	"username": "new_viewer",
	"password": "123456",
	"roles": "viewer"
}
```
- 角色约束：`roles` 只能是 `admin`、`inspector`、`viewer`
- 成功响应（201）：
```json
{
	"success": true,
	"user": {
		"id": 4,
		"username": "new_viewer",
		"roles": "viewer",
		"EmployeeID": "4"
	}
}
```
- 常见失败：
	- 400：`用户名和密码不能为空`
	- 400：`角色必须是 admin、inspector 或 viewer`
	- 400：`用户名已存在`
	- 401：未登录
	- 403：权限不足

5. 删除用户（仅管理员）
- 方法：`DELETE`
- 路径：`/users/:id`
- 权限：`admin`
- 路径参数：`id`（正整数，自增主键 UserID）
- 成功响应（200）：
```json
{
	"success": true,
	"deletedUserId": 4
}
```
- 常见失败：
	- 400：`用户ID不合法`
	- 404：`用户不存在`
	- 401：未登录
	- 403：权限不足
