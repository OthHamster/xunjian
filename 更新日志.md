# 更新日志

日期：2026-05-14

## 后端：数据库与空间能力

1. 接入 SpatiaLite 扩展加载与空间元数据初始化。
2. 数据库初始化新增业务表：users、route、risks、checkpoint、checkin。
3. route 表新增双几何列：
   - WGS84（SRID 4326，LINESTRING）用于地图展示
   - UTM（SRID 32651，LINESTRING）用于空间距离判定
4. 为 route.WGS84 与 route.UTM 创建空间索引。
5. 修复外键引用一致性：checkpoint.RouteID -> route.RouteID。

## 后端：路线功能与接口

1. 新增路线工具模块：backend/database_routers/route_utils.js。
2. 路线导入/更新逻辑：
   - 接收 WGS84 坐标数组
   - 写入 WGS84 列
   - 使用 ST_Transform 将 WGS84 路线转换后写入 UTM 列
3. 新增路线查询能力：
   - 获取单条路线（返回 WGS84/UTM 解析坐标）
   - 列出路线（含长度、点数）
4. 偏离监控逻辑调整：
   - 使用 ST_DWithin 在 UTM 坐标系下判断是否在缓冲范围内
   - 不再返回 distance，仅返回 isWithin（是否在范围内）
5. 缓冲距离默认值调整为 25 米。

## 后端：新增 REST API

1. GET /routes：获取路线列表。
2. GET /routes/:id：获取单条路线详情。
3. POST /routes：新增路线。
4. PUT /routes/:id：更新路线几何。
5. POST /routes/:id/check-location：偏离检测（返回是否在范围内）。

## 后端：依赖与其他变更

1. backend/package.json 新增 spatialite 依赖。
2. backend/package-lock.json 同步锁定 spatialite。
3. server.js 出现了 spatialite 的引入变更（const { Database } = require("spatialite")），当前主流程仍以 better-sqlite3 为主。

## 前端：管理端地图接入

1. web/my-app/package.json 新增 @amap/amap-jsapi-loader。
2. 新增管理端地图组件：
   - web/my-app/src/pages/admin/MapContainer.jsx
   - web/my-app/src/pages/admin/MapContainer.css
3. OnlineUsersPage 页面接入 MapContainer 展示地图。

## 文档更新

1. 新增 backend/SPATIALITE_GUIDE.md，包含 SpatiaLite 集成、空间函数与接口示例。
2. 巡检系统.md 补充路线/风险/检查点相关设计说明。

## 兼容与注意事项

1. UTM SRID 已固定为 32651（按列定义使用，不再动态推导）。
2. 旧数据库若已存在不同定义的 UTM 列，需要迁移或重建后再使用新逻辑。
3. 偏离检测当前只输出布尔结果，若后续需要距离值可再开启 ST_Distance 计算。
